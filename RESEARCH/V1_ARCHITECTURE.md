## Identity architecture (the amended contract)

The original contract was "no users table, ever." Memberships require one.
The amendment, stated plainly:

- **members table exists**: email, scrypt password hash, tier, created date.
  Email is the login key. **Display name optional** — members who leave it
  blank appear as "a member" on every post.
- **Posts link to member id** (required: "my journal" must know whose).
- **Research exports strip the linkage**: the corpus export (SUTRA pull)
  replaces member_id with a rotating anonymous key — correlation across
  posts survives, identity does not.
- **Free anonymous journal**: no account. A random device token
  (localStorage) identifies the journal. One post per day per token,
  IP-rate-limited in RAM. Private: readable only with the token.
  Honest caveat shown in UI: clearing browser data loses the journal.
  Optional name field, stored only if given.
- audit_record.mjs updated: it will now assert the *members table contains
  only email + hash + tier* — no names required, no tracking fields, and
  that no post row contains an email.

## Auth (v1: simple, correct, boring)

- Email + password. scrypt hashes (same as circle word). HTTP-only session
  cookie, 30-day expiry, SameSite=Lax. No OAuth, no magic links (v1.1).
- Signup creates a **free account**. Tier activation via **member code**:
  after Stripe checkout, Stripe emails you; you send the buyer a code
  (generated by `node makecode.mjs 33` → one-use code). They enter it once;
  account upgrades. Manual but AGENT-GATED — no money automation in v1.
  v1.1: Stripe webhooks automate this away.

## The bi-wheel ($66)

Natal chart from birth date/time/place (geocoding via a static city list
v1 — no external API, sovereignty). astronomy-engine computes natal
positions once, stored on the member row. The ring renders natal planets
as small glyphs on a second radius inside the wedges; the live Moon's
aspects to natal planets drive the personal aspectarian panel — the Moon
Transit Technique made visual. The clock itself never changes: same time,
same sky, one added layer of *their* sky.
